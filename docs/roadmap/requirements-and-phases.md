# Requirements And Phases

## Core Requirement

The resolver must remain the local decision engine for skill planning, locking, and materialization.

## Current Phase

The current phase is hardening the existing local flows:

- deterministic fresh planning
- strict policy enforcement
- lock replay
- cache, retry, logging, and telemetry improvements
- richer CLI and wizard ergonomics

## Next Phases

- better explanation quality for candidate selection
- broader policy-source management
- interface expansion through SDK and MCP surfaces
- controlled extensibility through plugins
